Google Sign-In Setup Guide

Enable "Sign in with Google" for the Kolamba platform

What is this? To allow users to sign in to Kolamba with their Google account, we need to register the application with Google. This is a one-time setup that takes about 5 minutes. Please follow the steps below and send us the credentials at the end.
1
Open Google Cloud Console

Go to the Google Cloud Console and sign in with your Google account:

https://console.cloud.google.com
Tip: If you don't have a Google Cloud account, it will be created automatically when you sign in with any Gmail account. No credit card is required — OAuth is a free service.
2
Create a New Project
  • Click the project dropdown at the top-left of the page (next to "Google Cloud")
  • Click New Project
  • Enter project name: Kolamba
  • Click Create and wait a few seconds for it to be ready
  • Make sure the new "Kolamba" project is selected in the top dropdown
3
Configure the OAuth Consent Screen

This tells Google what your app is called when users see the "Sign in with Google" popup.

  • In the left sidebar, go to APIs & Services → OAuth consent screen
  • Select External as the user type, then click Create
  • Fill in the required fields:
    FieldValue
    App nameKolamba
    User support emailYour email address
    Developer contact emailYour email address
  • Click Save and Continue
  • On the Scopes page — no changes needed, click Save and Continue
  • On the Test Users page — no changes needed, click Save and Continue
  • Click Back to Dashboard
4
Create OAuth Credentials

This generates the Client ID and Secret that we need to connect Kolamba to Google.

  • In the left sidebar, go to APIs & Services → Credentials
  • Click + Create Credentials at the top, then select OAuth client ID
  • Set Application type to Web application
  • Set Name to Kolamba Web
  • Under Authorized JavaScript origins, click + Add URI and add these two URLs:
    http://localhost:3000
    https://kolamba.vercel.app
  • Leave Authorized redirect URIs empty (not needed)
  • Click Create
Important: Make sure both URLs are added exactly as shown above, including http:// for localhost and https:// for the production URL. No trailing slashes.
5
Copy the Credentials

After clicking Create, a popup will appear showing your new credentials:

CredentialWhat it looks like
Client ID 123456789-xxxxxxxx.apps.googleusercontent.com
Client Secret GOCSPX-xxxxxxxxxxxxxxxxxx
A screenshot of this popup is the easiest way to share the credentials with us.

What We Need From You

  • 1
    Client ID
    Ends with .apps.googleusercontent.com
  • 2
    Client Secret
    Starts with GOCSPX-
What happens next? Once we receive these two values, we'll configure them in the Kolamba backend and frontend. The "Sign in with Google" button will then be active on the login page. No further action will be needed from your side.
