מה מועבר

שירות שיטת העברה מה מקבלים עלות חודשית
מאגר GitHub העברת מאגר קוד מקור מלא, היסטוריית שינויים, תהליכי CI/CD חינם
פרויקט Vercel העברת פרויקט אחסון צד לקוח, פריסה אוטומטית בעת דחיפת קוד, משתני סביבה חינם (Hobby) / $20 (Pro)
פרויקט Railway העברת פרויקט שרת API, בסיס נתונים PostgreSQL עם כל המידע ~$5–15 (לפי שימוש)
חשבון Cloudinary מסירת פרטי גישה אחסון תמונות ומדיה, כל הקבצים שהועלו חינם (25GB) / $89 (Plus)
פרויקט Google Cloud העברת בעלות פרטי התחברות OAuth עבור כניסה באמצעות Google חינם
דומיין העברת רשם בעלות על דומיין kolamba.com / kolamba.org ~$12/שנה

שלבי ההעברה

שלב א׳: הקמת חשבונות (הלקוח/ה)

יש ליצור חשבונות בפלטפורמות הבאות לפני שיחת ההעברה. לכולן יש תוכנית חינמית.

1
יצירת חשבון GitHub
יש להיכנס ל-github.com/signup — המאגר יועבר לחשבון זה. אם כבר יש לך חשבון, שתפ/י את שם המשתמש.
2
יצירת חשבון Vercel
יש להיכנס ל-vercel.com/signup — מומלץ להירשם עם חשבון GitHub לאינטגרציה חלקה. תוכנית Hobby (חינם) מספיקה להתחלה.
3
יצירת חשבון Railway
יש להיכנס ל-railway.app — מומלץ להירשם עם חשבון GitHub. יש להוסיף אמצעי תשלום (נדרש לאחסון בסביבת ייצור, חיוב לפי שימוש ~$5–15/חודש).

שלב ב׳: ביצוע ההעברה (יחד)

4
העברת מאגר GitHub
Drishti תתחיל את ההעברה מ-github.com/luvchikavi/kolamba ← לחשבון שלך. תקבל/י אימייל לאישור ההעברה. כל הקוד, הענפים וההיסטוריה נשמרים.
5
העברת פרויקט Vercel
Drishti תעביר את פרויקט Vercel לחשבון שלך. לאחר הקבלה: יש לחבר מחדש את מאגר GitHub (שכעת תחת החשבון שלך) בהגדרות הפרויקט ב-Vercel, ולאמת שמשתני הסביבה מוגדרים (ראו סעיף משתני סביבה להלן).
6
העברת פרויקט Railway
Drishti תעביר את פרויקט Railway (שרת + בסיס נתונים) לחשבון שלך. בסיס הנתונים וכל המידע שבו מועברים עם הפרויקט. יש לאמת את משתני הסביבה לאחר הקבלה.
7
העברת Cloudinary ו-Google OAuth
Drishti תמסור את פרטי הגישה לחשבון Cloudinary (או תסייע ביצירת חשבון חדש). עבור Google OAuth, Drishti תוסיף אותך כבעלים בפרויקט Google Cloud, ולאחר מכן תסיר את עצמה.
8
העברת דומיין (אם רלוונטי)
העברת דומיין דרך הרשם. רשומות DNS יצטרכו להפנות לפריסות Vercel ו-Railway שלך. Drishti תסייע בהגדרת DNS במהלך ההעברה.

שלב ג׳: אבטחה (הלקוח/ה)

9
החלפת כל הסודות
לאחר ההעברה, יש ליצור ערכים חדשים עבור SECRET_KEY וכל מפתח API שתרצו להחליף. יש לעדכן אותם במשתני הסביבה ב-Railway וב-Vercel. כך רק לך תהיה גישה.

מדריך משתני סביבה

Railway – צד שרת (כבר מוגדר — יש לאמת לאחר ההעברה)

# Database (auto-managed by Railway)
DATABASE_URL=postgresql+asyncpg://...

# Security — generate a new random 64-char string after transfer
SECRET_KEY=<generate-new-secret>
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=15
REFRESH_TOKEN_EXPIRE_DAYS=7

# CORS — update with your domain
CORS_ORIGINS=https://kolamba.vercel.app

# Cloudinary (media uploads)
CLOUDINARY_CLOUD_NAME=<from-cloudinary-dashboard>
CLOUDINARY_API_KEY=<from-cloudinary-dashboard>
CLOUDINARY_API_SECRET=<from-cloudinary-dashboard>

# Google OAuth
GOOGLE_CLIENT_ID=<from-google-cloud-console>
GOOGLE_CLIENT_SECRET=<from-google-cloud-console>

# Email (optional — needed for sending emails)
RESEND_API_KEY=<from-resend.com-if-needed>

Vercel – צד לקוח (כבר מוגדר — יש לאמת לאחר ההעברה)

NEXT_PUBLIC_API_URL=https://kolamba-api-production.up.railway.app
NEXT_PUBLIC_GOOGLE_CLIENT_ID=<same-google-client-id-as-backend>

כיצד עובדת הפריסה

פריסה אוטומטית (ללא מאמץ)

גם צד הלקוח וגם צד השרת נפרסים אוטומטית כשקוד נדחף לענף main ב-GitHub. אין צורך להריץ פקודות — פשוט דחפו את הקוד.

  • צד לקוח: Vercel מזהה את הדחיפה, בונה את Next.js, ופורס תוך ~60 שניות.
  • צד שרת: Railway מזהה את הדחיפה, בונה את מכולת Docker, מריץ מיגרציות לבסיס הנתונים, ופורס תוך ~2 דקות.

מיגרציות בסיס נתונים

שינויי סכימה בבסיס הנתונים מנוהלים על ידי Alembic (כלי מיגרציה של Python). המיגרציות רצות אוטומטית בכל פריסת Railway דרך start.sh. אין צורך בניהול ידני של בסיס הנתונים.

רשימת בדיקות לאחר ההעברה

  • מאגר GitHub נגיש תחת החשבון שלך
  • ניתן לשכפל ולדחוף למאגר
  • פרויקט Vercel מופיע בלוח הבקרה שלך
  • Vercel מחובר למאגר GitHub ופורס אוטומטית
  • צד הלקוח נטען בכתובת kolamba.vercel.app
  • פרויקט Railway מופיע בלוח הבקרה שלך
  • בדיקת תקינות צד השרת מחזירה “healthy”
  • התחברות והרשמה עובדות באתר
  • התחברות באמצעות Google OAuth עובדת
  • העלאת תמונות עובדת (Cloudinary)
  • בסיס הנתונים מכיל את כל המידע הקיים (בדקו בלוח ניהול)
  • מפתח SECRET_KEY חדש נוצר והוגדר

פעולות נפוצות

משימה כיצד לבצע
צפייה בלוגים של השרת לוח בקרה Railway ← פרויקט Kolamba ← שירות Backend ← לשונית Logs
צפייה בהיסטוריית פריסות לוח בקרה Vercel ← kolamba ← לשונית Deployments
גישה לבסיס הנתונים לוח בקרה Railway ← שירות PostgreSQL ← לשונית Data (או שימוש במחרוזת חיבור עם pgAdmin)
עדכון משתני סביבה Railway: שירות ← לשונית Variables. Vercel: פרויקט ← Settings ← Environment Variables
ביטול פריסה (Rollback) Vercel: Deployments ← לחיצה על פריסה קודמת ← Promote to Production
הוספת דומיין מותאם Vercel: Settings ← Domains ← Add ← עקבו אחרי הוראות DNS
אישור אמנים ממתינים התחברות כמנהל ← לוח ניהול ← אמנים ← שינוי סטטוס ל-“active”
