1. Platform Overview

Kolamba is a web platform that connects performing artists (talents) with communities (hosts) for live events, concerts, workshops, and tours. The platform handles discovery, booking, quoting, messaging, and tour planning.

Discovery
Hosts browse and search for talents by category, price, and location. Talents are displayed in a card grid with filters.
Booking & Quotes
Hosts send booking requests to talents. Talents submit price quotes. Hosts approve, decline, or request changes.
Communication
Each booking creates a conversation thread. Both parties exchange messages, share venue details, and coordinate logistics.
Tour Management
Talents create tours with dates, stops, constraints, and route planning. Bookings auto-generate tour stops.

3. Quote & Booking Flow

The quote flow is the core booking process. It connects hosts who want to book a performance with talents who provide pricing.

Host Side — Sending a Booking Request

1
Browse & Select a Talent
Host logs in → clicks "Browse Talents" from the Host Dashboard. Selects a talent profile and clicks "Send Booking Request".
2
Fill the Booking Form
Host fills in the booking details: Event type (Concert, Workshop, Lecture, etc.), Preferred date, Location & venue type, Expected audience size, Budget range, and Special requirements / notes. Submits the form and sees a "Quote Request Sent!" confirmation.
3
Wait for Quote
The booking appears in the Host Dashboard → Quotes tab with status Pending. Also visible in the Messages tab with a conversation thread. Host can send messages to the talent while waiting.
4
Receive & Respond to Quote
When the talent submits a quote, status changes to Quote Received. A quote card appears in the messages showing the quoted amount, what's included, and date submitted. Host has three options: Approve → status becomes Approved, Request Changes → status returns to Pending, Decline (with reason) → status becomes Declined.

Talent Side — Receiving & Quoting

1
Receive Booking Request
Talent gets a notification: "[Host] has sent you a booking request for [location]". Booking appears in Talent Dashboard → Requests tab with status Pending. The Messages tab shows a conversation with event details: event type, date, location, audience size, budget, and notes.
2
Submit a Quote
In the Messages view, a "Submit Quote" card appears. Talent fills in: Amount (required) and What's included (optional, e.g., "2-hour performance, sound equipment, travel costs"). Clicks "Submit Quote" → status changes to Quote Sent. An automatic message is added: "Quote submitted: $2,500.00 — What's included: ..."
3
Wait for Response
The quote card shows a read-only summary with "Waiting for host response". Talent can continue sending messages in the conversation thread.
4
Receive Response
If approved: Green banner — "Booking Approved!" → Status becomes Approved.
If changes requested: Status returns to Pending → Talent can submit a revised quote.
If declined: Red banner with host's reason → Status becomes Declined.

4. Communication & Messaging

Every booking automatically creates a conversation thread between the host and talent. Messaging is integrated directly into the booking workflow.

How It Works

Automatic Thread Creation
When a host sends a booking request, a conversation is automatically created. Both parties can message each other immediately.
Inbox View
The Messages tab shows all conversations as a list with: contact name, last message preview, message count, booking status badge, and date.

Host Messaging Features

Feature Description
Quote Review Card When a talent submits a quote, a card appears showing the amount, notes, and Approve / Decline / Request Changes buttons
Booking Details Card Shows artist name, event date, location, budget, status, and notes in each conversation
Venue Information Panel Editable form with 16 fields: facility size, venue type, stage dimensions, sound system, lighting, camera availability, green room, catering, parking, accessibility, Wi-Fi, load-in access, expected attendance, audience type, and additional notes
Activity Log Chronological message thread showing all messages with timestamps and sender role badges
Send a Note Text area at the bottom to send messages to the talent

Talent Messaging Features

Feature Description
Quote Submission Form When a booking is Pending, a form appears to enter amount and "what's included" details
Quote Status Display After submitting, shows the quote summary with "Waiting for host response" indicator
Venue Info Button Read-only panel showing the host's venue details (talent cannot edit these)
Message Bubbles Own messages appear right-aligned (blue), other's messages left-aligned (gray), with timestamps and sender names

Venue Information Fields (Host-Editable)

Hosts can share detailed venue information with talents through the messaging panel. This includes:

  • Text fields: Facility size, venue type, stage dimensions, expected attendance, audience type, parking, accessibility, load-in access, additional notes
  • Yes/No fields: Sound system, lighting, camera available, green room, catering, Wi-Fi

5. Tour Management

Talents can plan and manage tours with multiple stops, constraints, and automatic booking integration.

Creating a Tour

1
Start a New Tour
From the Talent Dashboard, click "Start a Tour". Fill in the tour details: Tour Name, Region / Area, Start & End Dates, Price per Show, Minimum Budget, and Description.
2
Set Tour Constraints (Optional)
Expand the "Tour Constraints" section to set rules: Max Travel Hours between stops, Min/Max Shows per Week, Rest Day Rule (e.g., every Wednesday), Minimum Net Profit, and Visa Status.
3
Publish the Tour
Click "Publish Tour" to create the tour. If you have existing bookings, they can be attached and will auto-generate tour stops.

Tour Detail Page

Each tour has a detail page showing info cards (dates, confirmed shows, price per show, confirmed revenue), tour rules, and a stops table.

Tour Stops

Stop Type Color Description
Confirmed Green An anchor booking — confirmed show with venue and date
Open Yellow Available date slot for hosts to book
Rest Day Gray Blocked day per rest rules (no bookings)
Inquiry Blue Pending discussion / negotiation with a host
Recommended — System-suggested stop based on tour algorithm scoring

Stops Table Columns

Column Description
Date Scheduled date for the stop (e.g., Sat, Mar 15)
City / Venue Location and venue name (TBD for open slots)
Status Confirmed, Open, Rest Day, Inquiry, or Recommended
Travel Estimated travel time from previous stop
Est. Revenue Expected payment for confirmed/inquiry stops

Booking → Stop Auto-Creation

Automatic Integration

When a booking is added to a tour, the system automatically creates a linked TourStop. This happens in three ways:

  • Adding a booking to an existing tour
  • Creating a tour with existing booking IDs
  • Approved join request — auto-creates a stop for the new booking

Tour Constraints Reference

Constraint Description Example
Max Travel Hours Maximum driving time between consecutive stops 4 hours
Min Shows/Week Minimum number of performances per week 2
Max Shows/Week Maximum number of performances per week 5
Rest Day Rule Recurring rest day (blocked from bookings) Every Wednesday
Min Net Profit Minimum net profit target for the tour $8,000
Visa Status Travel visa requirements Not Required

6. Host Dashboard Guide

The host dashboard is the control center for community managers. Access it at /dashboard/host.

Tab Description
Overview Stats summary: active bookings, pending quotes, messages, and recent activity
Events / Quotes All booking requests with status badges. Manage quotes — approve, decline, or request changes
Messages Conversation threads with talents. View booking details, submit venue info, review quotes, and exchange messages
Discover Browse talents with filters (category, price, distance). Interest-based matching shows compatible artists
Touring Artists See artists currently on tour near your location
Settings Edit community profile: name, location, audience size, event types, contact details

7. Talent Dashboard Guide

The talent dashboard is the hub for artists to manage their career on Kolamba. Access it at /dashboard/talent.

Tab Description
Overview Stats summary: upcoming shows, pending requests, total bookings, and recent activity
Requests Incoming booking requests from hosts with event details, dates, and budget info
Messages Conversation threads with hosts. Submit quotes, view venue info, and exchange messages
Tours Create and manage tours. View tour detail pages with stops table, rules, and revenue tracking
Settings Edit artist profile: name (EN/HE), bio, pricing (single/tour), categories, city, social links, profile photo

8. Admin Dashboard Guide

The admin dashboard is available to superusers. The admin sidebar is always visible at /dashboard/admin and persists when viewing Host or Talent dashboards.

Admin Sidebar Navigation

Link Description
Overview Platform stats, growth charts (line), booking status breakdown (pie), artists by category (bar)
Users Full user management table with search, role/status filters. Per-user actions: Edit (modal with profile details), Delete
Talents Talent management cards with status pills (All, Pending, Active, Rejected, Inactive). Actions: Approve, Reject (with reason), Deactivate, Toggle Featured

"View As" Quick Links

Link Destination Purpose
Talent Dashboard /dashboard/talent View the platform from a talent's perspective
Host Dashboard /dashboard/host View the platform from a host's perspective

User Management Actions

Edit User
Click any user row or the "Edit" button. Opens a modal with: name, role, status, active toggle, and profile-specific fields (artist profile, community profile, or agent managed artists).
Delete User
Red "Delete" button on each user row (except superusers and already-deleted users). Shows confirmation dialog. Sets user to inactive/deleted status.

Talent Management Actions

Approve
For pending or rejected talents. Sets status to "active" and makes the profile visible on the public site.
Reject
For pending talents. Opens a modal to select a reason (Incomplete Profile, Insufficient Portfolio, Not a Relevant Fit, Duplicate Account). Reason is visible to the talent.
Deactivate
For active talents. Sets status to "inactive" and hides the profile from public listings.
Toggle Featured
Star icon on each talent card. Featured talents appear in the homepage carousel.

9. Notifications

The platform sends in-app notifications for key events. A notification bell with an unread count badge appears in the header.

Event Who Gets Notified Message
New booking request Talent "[Host] has sent you a booking request for [location]"
Quote submitted Host "[Talent] has submitted a quote of $X for your booking"
Quote approved Talent "[Host] has approved your quote of $X!"
Quote declined Talent "[Host] has declined your quote. Reason: [reason]"
Changes requested Talent "[Host] has requested changes to your quote: [reason]"

Notification Features

  • Unread count badge on the bell icon in the header
  • Click to view all notifications in a dropdown
  • Mark individual notifications as read
  • "Mark all as read" button
  • Auto-notify on new bookings and quote events

Booking / Quote Statuses

Status Badge Meaning
Pending Pending Booking request sent, waiting for talent to submit a quote

Approved Approved Host accepted the quote — booking is confirmed
Declined Declined Host rejected the quote (reason provided)

User Statuses

Inactive Inactive Account deactivated — profile hidden from public listings
